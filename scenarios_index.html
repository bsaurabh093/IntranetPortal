GAIL- GMS Implementation

Scenarios in GMS

Objective

This document outlines various possible scenarios that can occur during a daily or fortnightly GMS execution. This would also detail the various steps to be followed in case something needs to be altered in between this execution.

Flow Chart Legend:

In the flow chart, the boxes indicated as Green refer to the business process steps already executed.

In the flow chart, the box indicated as Red refers to the business process step where the need to change / update was identified.

Scenario 1: Change in Nominated Qty. after Nomination Upload for Single Day / Multiple Days

Description: Nominations have been uploaded for a single Day and then there is a change to the Nominated Quantity for the uploaded contracts.

Steps to be followed:

  • Create a fresh nomination upload excel sheet with the delta changes. By doing this, there is a log for the changes made. Multiple Excel sheets would be needed per Gas Day wherein there is a change.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in Display Nominations(O4NSN) if the contractual quantities have been updated

Scenario 2: Change in Measurements (Volume, GCV, NCV) before Posting in Measurement Editor for single / multiple days

Description:
Nominations Measurements have been uploaded for a single day / multiple days and then there is a change to the Measurement Values that have been uploaded.

Steps to be followed:

  • Prepare the updated measurement excel sheet for all dates together.
  • Upload the measurement sheet using Measurement Upload (YRXU002).
  • Check in the Measurement Editor (YRXU003 for all dates), if the uploaded values are showing up as needed.

Scenario 3: Change in Nomination Quantity after upload of Measurements

Description: Nominations Measurements have been uploaded for a single day / multiple days and then there is a change to the Contractual Nominated Quantity for the already uploaded contracts.

Steps to be followed:

  • Create a fresh nomination upload excel sheet with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated
  • Note: No Need to upload the Measurement again if measurements have already been uploaded.

Scenario 4: Change in Measurements after posting in Measurement Editor

Description: Measurement Editor has been posted and there is a change to the uploaded Qty (Volume, GCV, NCV) for a single / multiple days.

Steps to be followed:

  • Prepare the updated measurement excel sheet for all dates together.
  • Upload the measurement sheet using Measurement Upload (YRXU002).
  • Check in the Measurement Editor (YRXU003 for all dates), if the uploaded values are showing up as needed.
  • Post in Measurement Editor again for the single day so that allocations are triggered.

Scenario 5: Change in Nomination quantity after posting in Measurement Editor

Description: Measurement Editor (Allocations) has been posted and then there is a change to the Nominated Qty for single or multiple days.

Steps to be followed:

  • Create multiple nomination upload excel sheets (per gas day) with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated.
  • Re-Run Measurement Editor (YRXU003) for the gas days which have been updated.

Scenario 6: Change in Nomination Quantity after running Daily Quantity Adjustments (OD1)

Description: Daily Quantity Adjustments (OD1) has been performed and then there is a change to the Nominated Qty for single or multiple days.

Steps to be followed:

  • Create multiple nomination upload excel sheets (per gas day) with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated.
  • Check if Measurement Values are the same or need to be updated.
  • Re-Run Measurement Editor (YRXU003) for the gas days which have been updated.
  • Re-Run Daily Quantity Adjustments (OD1) for the gas dates where there is an update.

Scenario 7: Change in Uploaded Measurements after running Daily Quantity Adjustments (OD1)

Description: Daily quantity adjustments have been processed and then there is a need to change the Measurement Values for single or multiple days.

Steps to be followed:

  • Prepare the updated measurement excel sheet for all dates together.
  • Upload the measurement sheet using Measurement Upload (YRXU002).
  • Check in the Measurement Editor (YRXU003 for all dates), if the uploaded values are showing up as needed.
  • Post in Measurement Editor again for the multiple days so that fresh allocations are triggered.
  • Re-Run Daily Quantity Adjustments (OD1) for the gas dates where there is an update.

Scenario 8: Change in Nominated Qty after posting deliveries from Allocation Editor

Description: Deliveries have been posted for single / multiple days (using Allocation Editor) and then there is a change to the Nomination Qty.

Steps to be followed:

  • Reverse all the Deliveries manually by using the transaction O4TENREV.[Search for the Tickets by using the transaction O4NSN]
  • Create a fresh nomination upload excel sheet with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated
  • Re-Run Measurement Editor again for the multiple dates so that fresh allocations are triggered.
  • Re-Run Daily Quantity Adjustments (OD1) for the multiple dates where there is an update.
  • Re-Run Allocation Editor (YRXU001) and post deliveries for all those customers where nominations have changed.

Scenario 9: Change in Measurements after posting deliveries from Allocation Editor

Description: Deliveries have been posted for a multiple days (using Allocation Editor) and then there is a change to the Measurement Values for multiple days.

Steps to be followed:

  • Reverse all the Deliveries(for business locations where there is a change) manually by using the transaction O4TENREV.[Search for the Tickets by using the transaction O4NSN]
  • Prepare the updated measurement excel sheet for all dates together.
  • Upload the measurement sheet using Measurement Upload (YRXU002).
  • Check in the Measurement Editor (YRXU003 for all dates), if the uploaded values are showing up as needed.
  • Post in Measurement Editor again for the multiple days so that fresh allocations are triggered.
  • Re-Run Measurement Editor again for the multiple dates so that fresh allocations are triggered.
  • Re-Run Daily Quantity Adjustments (OD1) for the multiple dates where there is an update.
  • Re-Run Allocation Editor (YRXU001) and post deliveries for all those customers where nominations have changed.

Scenario 10: Change in Nominations after Billing is completed

Description: Billing has been completed for single or multiple customers and then there is a change to the Nomination Qty for them.

Steps to be followed:

  • Cancel the Bill Document generated using the Transaction VF11.
  • Reverse all the Deliveries manually by using the transaction O4TENREV.[Search for the Tickets by using the transaction O4NSN]
  • Create a fresh nomination upload excel sheet with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated
  • Re-Run Measurement Editor again for the multiple dates so that fresh allocations are triggered.
  • Re-Run Daily Quantity Adjustments (OD1) for the multiple dates where there is an update.
  • Re-Run Allocation Editor (YRXU001) and post deliveries for all those customers where nominations have changed.

Scenario 11: Change in Measurements after Billing is completed

Description: Billing has been completed for single or multiple customers and then there is a change to the Measurements captured for these customers for single / multiple days

Steps to be followed:

  • Cancel the Billing Document generated using the Transaction VF11.
  • Reverse all the Deliveries for the customers for the specific dates, manually by using the transaction O4TENREV.[Search for the Tickets by using the transaction O4NSN]
  • Create a fresh nomination upload excel sheet with the delta changes. By doing this, there is a log for the changes made.
  • Update the nomination qty. for the respective contract and upload the excel sheet using the transaction YRXR001.
  • Check in O4NSN if the contractual quantities have been updated
  • Re-Run Measurement Editor again for the multiple dates so that fresh allocations are triggered.
  • Re-Run Daily Quantity Adjustments (OD1) for the multiple dates where there is an update.
  • Re-Run Allocation Editor (YRXU001) and post deliveries for all those customers where nominations have changed.
